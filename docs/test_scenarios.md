# star_record 실험 시나리오

> 실제 스타크래프트 리마스터를 설치 후, star_record의 각 기능을 검증하는 단계별 가이드.

---

## 0. 사전 준비

### 0-1. 스타크래프트 설치
1. Battle.net 런처 설치 (https://battle.net)
2. Battle.net에서 스타크래프트: 리마스터 설치 (무료 / 리마스터 유료)
3. 한 번 실행하여 정상 동작 확인 후 종료

### 0-2. 리플레이 폴더 위치 확인
스타크래프트 실행 후 게임을 한 판 플레이(또는 싱글플레이)하고 리플레이 저장.
이후 아래 경로 중 리플레이가 저장된 곳을 찾는다:

```
일반적인 경로:
  C:\Users\<유저>\Documents\StarCraft\Maps\Replays\
  C:\Users\<유저>\Documents\StarCraft Remastered\Replays\
  C:\Program Files (x86)\StarCraft\Maps\Replays\

확인 방법:
  1) 게임 내에서 리플레이 저장 후 탐색기에서 검색: *.rep
  2) 또는 게임 설정 > 리플레이 경로 확인
```

> 찾은 경로를 메모해 둔다. 이후 `<REPLAY_DIR>` 로 표기한다.

### 0-3. 스타크래프트 실행파일 경로 확인
```
일반적인 경로:
  C:\Program Files (x86)\StarCraft\StarCraft.exe
  C:\Program Files (x86)\StarCraft\x86\StarCraft.exe

확인 방법:
  Battle.net 런처 > 스타크래프트 > 설정(톱니바퀴) > 설치 경로
```

> 찾은 경로를 메모해 둔다. 이후 `<SC_PATH>` 로 표기한다.

### 0-4. Python 패키지 설치
```bash
cd D:\pypy\star_record
pip install -r requirements.txt
```

## 1. 초기 설정 테스트

### 시나리오 1-1: 경로 설정
```bash
python main.py set-sc-path "<SC_PATH>"
python main.py set-replay-dir "<REPLAY_DIR>"
```

**예상 결과:**
- "스타크래프트 경로 설정: ..." 출력
- "리플레이 폴더 설정: ..." 출력
- config.json에 두 경로가 저장됨

**검증:**
```bash
type config.json
```
→ `starcraft_path`와 `replay_dir`에 값이 들어있어야 한다.

---

### 시나리오 1-2: 본인 닉네임 등록
```bash
python main.py set-name <내_배틀넷_닉네임>
```

**예상 결과:**
- "본인 닉네임 등록: ..." 출력
- config.json의 `my_names`에 추가됨

## 2. 리플레이 가져오기 테스트

### 시나리오 2-1: 싱글플레이 리플레이 생성
1. 스타크래프트 실행
2. 싱글플레이 > 커스텀 게임 > 컴퓨터 상대로 1판 플레이
3. 게임 종료 후 리플레이 저장 (게임 끝나면 자동 저장되는지 확인)
4. `<REPLAY_DIR>` 폴더에 .rep 파일이 생겼는지 확인

**확인 사항:**
- [ ] .rep 파일이 생성되는가
- [ ] 파일명 형식은 어떤가 (날짜 포함 여부, 플레이어 이름 포함 여부)
- [ ] 파일명 형식을 메모해 둔다 (파서 호환 여부 판단용)

---

### 시나리오 2-2: 멀티플레이 리플레이 생성
1. 스타크래프트 실행 > 멀티플레이 > Battle.net
2. 커스텀 게임에서 다른 사람과 1v1 플레이
3. 게임 종료
4. `<REPLAY_DIR>` 에 .rep 파일 생성 확인

**확인 사항:**
- [ ] 멀티플레이 리플레이도 같은 폴더에 저장되는가
- [ ] 파일명에 상대 닉네임이 포함되는가
- [ ] 래더(랭크) 게임도 리플레이가 저장되는가

---

### 시나리오 2-3: 리플레이 일괄 가져오기
```bash
python main.py import "<REPLAY_DIR>"
```

**예상 결과:**
- "N개의 리플레이 파일 발견" 로그
- "N개 새로 저장됨" 출력
- 본인 닉네임 자동 추론 시도

**검증:**
```bash
python main.py records
```
→ 상대별 전적 표가 출력되어야 한다.

**확인 사항:**
- [ ] 파싱이 성공하는가 (에러 없이 완료)
- [ ] 플레이어 이름이 정확히 추출되는가
- [ ] 승패 판정이 맞는가
- [ ] 종족이 정확한가
- [ ] 맵 이름이 추출되는가
- [ ] 게임 시간이 정확한가

---

### 시나리오 2-4: 파싱 실패 시 확인
만약 파싱 에러가 발생하면:
```bash
python main.py -v import "<REPLAY_DIR>"
```
→ `-v` 옵션으로 상세 로그를 확인하고, 어떤 파일에서 에러가 나는지 기록.

**가능한 원인:**
- 리마스터에서 리플레이 포맷이 변경됨
- 파일명 규칙이 다름 (날짜 추출 실패)
- 플레이어 이름 파싱 오프셋이 다름

## 3. 폴더 감시 (watch) 테스트

### 시나리오 3-1: watch 모드 + 게임 플레이
1. 터미널에서 watch 시작:
   ```bash
   python main.py watch "<REPLAY_DIR>"
   ```
2. 스타크래프트 실행 후 1판 플레이
3. 게임 종료

**예상 결과:**
- 게임 종료 후 몇 초 내에 "새 리플레이 감지" 로그
- 전적 정보 콘솔 출력
- Windows 토스트 알림 표시

**확인 사항:**
- [ ] 새 리플레이를 감지하는가
- [ ] 감지까지 걸리는 시간은 (즉시 / 몇 초 / 안 됨)
- [ ] 토스트 알림이 뜨는가
- [ ] 이미 가져온 리플레이를 중복 처리하지 않는가

---

### 시나리오 3-2: LastReplay.rep 확인
스타크래프트가 `LastReplay.rep` 파일을 별도로 관리하는지 확인:
```
<REPLAY_DIR> 또는 상위 폴더에 LastReplay.rep가 있는지 탐색
```

**확인 사항:**
- [ ] LastReplay.rep가 존재하는가
- [ ] 게임 시작 시 생성/갱신되는가, 아니면 종료 후에 생성되는가
- [ ] LastReplay.rep 경로: ____________________

> 게임 **시작** 시 갱신된다면, 게임 시작과 동시에 상대 정보를 미리 알 수 있다.
> 게임 **종료** 후에만 생성된다면, 현재 동작과 동일 (종료 후 전적 알림).

## 4. 런처 모드 테스트

### 시나리오 4-1: 런처로 스타크래프트 실행
```bash
python main.py launch
```
또는 `StarRecord_Launcher.bat` 더블클릭.

**예상 결과:**
- "리플레이 감시 시작됨" 로그
- 스타크래프트가 실행됨
- 게임 플레이 후 종료하면 전적 알림
- 스타크래프트 종료 후 프로그램도 종료

**확인 사항:**
- [ ] 스타크래프트가 정상 실행되는가
- [ ] Battle.net 런처를 거쳐서 실행되는 경우 동작이 다른가
  - SC가 Battle.net을 통해 실행되면, `StarCraft.exe` 직접 실행이 안 될 수 있음
  - 이 경우 Battle.net 런처 경로로 설정해야 할 수 있음
- [ ] SC 종료 후 프로그램이 함께 종료되는가
- [ ] SC 종료 감지까지 시간이 얼마나 걸리는가

### 시나리오 4-2: Battle.net 경유 문제
Battle.net을 통해서만 실행 가능한 경우:
1. `StarCraft.exe`를 직접 실행했을 때 Battle.net이 열리는지 확인
2. 열린다면 `set-sc-path`에 Battle.net 런처 경로를 넣어야 할 수 있음
3. 또는 launch 모드 대신 daemon 모드를 사용

> 이 결과에 따라 launcher.py 수정이 필요할 수 있다.

## 5. 데몬 모드 테스트

### 시나리오 5-1: 데몬 실행 → 게임 시작 → 게임 종료
1. 터미널에서 데몬 시작:
   ```bash
   python main.py daemon
   ```
2. "스타크래프트 실행을 감시하고 있습니다..." 메시지 확인
3. 스타크래프트를 별도로 실행 (Battle.net에서 정상적으로)
4. 게임 1판 플레이 후 종료
5. 스타크래프트 완전 종료

**예상 결과:**
- SC 시작 시: "[*] 스타크래프트 감지 - 리플레이 감시 시작" + 토스트 알림
- 게임 종료 시: 전적 알림
- SC 종료 시: "[*] 스타크래프트 종료 - 리플레이 감시 중지" + 토스트 알림
- 다시 대기 상태로 돌아감

**확인 사항:**
- [ ] `StarCraft.exe` 프로세스 이름이 정확한가 (Task Manager에서 확인)
  - 실제 프로세스 이름: ____________________
  - launcher.py의 `SC_PROCESS_NAMES`와 일치하는가
- [ ] SC 시작 감지가 되는가
- [ ] SC 종료 감지가 되는가
- [ ] 감지 → 다시 대기 → 재감지 사이클이 정상 동작하는가

### 시나리오 5-2: 프로세스 이름 확인
게임 실행 중 다른 터미널에서:
```bash
tasklist | findstr -i starcraft
```
→ 실제 프로세스 이름을 확인하고, `launcher.py`의 `SC_PROCESS_NAMES`와 비교.
일치하지 않으면 수정 필요.

## 8. 엣지 케이스

### 시나리오 8-1: 리플레이 자동 저장 vs 수동 저장
- [ ] 게임 종료 후 리플레이가 자동 저장되는가, 수동으로 저장해야 하는가
- [ ] 자동 저장이면: 저장 경로가 `<REPLAY_DIR>` 맞는가
- [ ] 수동 저장이면: 사용자가 저장을 안 하면 전적 기록 불가 (제한사항)

### 시나리오 8-2: 게임 비정상 종료
- [ ] 게임 도중 Alt+F4로 종료하면 리플레이가 저장되는가
- [ ] 연결 끊김(디스커넥)일 때 리플레이가 저장되는가

### 시나리오 8-3: 팀 게임 (2v2, 3v3)
- [ ] 팀 게임 리플레이도 파싱이 되는가
- [ ] 플레이어가 3명 이상일 때 에러가 나지 않는가
- [ ] (현재 파서는 1v1 기준, 팀 게임 지원은 추후 검토)

### 시나리오 8-4: Use Map Settings (유즈맵)
- [ ] 유즈맵 리플레이도 파싱이 되는가
- [ ] 유즈맵은 전적에서 제외해야 할 수 있음 (game_type으로 필터링 가능)

## 10. 문제 발생 시 수집할 정보

버그가 있으면 다음을 기록:
1. 에러 로그: `python main.py -v <명령어>` 로 상세 로그 확인
2. 리플레이 파일: 문제 되는 .rep 파일을 `replays/samples/`에 보관
3. 프로세스 이름: `tasklist | findstr -i starcraft` 결과
4. 리플레이 파일명: 실제 저장되는 형식 메모
5. config.json 내용
